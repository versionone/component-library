## Supporting Visual

A list item may optionally contain a supporting visual. These are not limited to simply icons, but may also be interactive components. When using an interactive component as the supporting visual, the onClick event of the supporting visual will **not** trigger the `ListItem` primary action (its `onClick` event handler).

## Item Actions

List items are clickable; which represents the primary action. Items may also contain a single, ancillary action; a secondary action.

> Clicking a secondary action will **not** trigger the primary action (`onClick` event handler on the `ListItem`).

## Keyboard Navigation

List items with primary actions are focus-able. When a list item is focused, then tabbing will focus the list item's supporting visual and then secondary action. Pressing up/down arrow keys will move the focus to the next list item.

> All list items are focus-able, even if they do not contain a primary action.

## List Item Content Lines

A `ListItem` may have up to 3 distinct content values; `primary`, `secondary`, `tertiary`. Each value will render on its own line. Content that is too long will be [clamped](/components/Clamp) (to a single line only).

> A list item text's values can contain click event handlers; triggering one will **not** trigger the click event handler on the `ListItem`.
